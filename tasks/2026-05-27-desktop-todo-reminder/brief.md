# Product Brief

## Problem

用户需要一个以电脑端可靠提醒为核心的待办工具，同时希望手机端也能查看、添加和修改待办事项。普通网页待办工具在桌面后台提醒、系统托盘、开机启动和本地通知可靠性上不够像桌面应用；而直接做完整云同步和移动端原生应用又会让第一版范围过大。

第一版应先交付一个可本地运行、可验证的桌面待办提醒工具：电脑端负责本地数据、提醒调度和系统通知；手机端通过同一 Wi-Fi 下的 Web 页面访问电脑端数据，用扫码或配对码建立临时授权。

## User

主要用户是个人使用者：在电脑前工作时需要可靠待办提醒，也希望离开电脑或拿起手机时能快速查看、添加和完成任务。

## Goal

构建一个桌面应用优先的待办提醒工具，支持电脑端原生提醒、系统托盘、后台运行和本地任务管理，并提供手机端 Web 入口用于同一局域网内同步查看和编辑。

## Non-goals

- 第一版不做云同步、账号体系或远程服务器部署。
- 第一版不做 iOS / Android 原生应用。
- 第一版不做多人协作、团队空间或权限管理。
- 第一版不做复杂日历集成、自然语言识别或 AI 排程。
- 第一版不要求手机端后台推送；提醒由电脑端负责触发。

## Success Criteria

- 电脑端可以创建、编辑、完成、删除待办，并设置提醒时间。
- 电脑端可以在后台运行，到点触发系统通知。
- 电脑端提供系统托盘入口，可以打开主窗口、暂停提醒和退出应用。
- 电脑端使用本地 SQLite 保存任务，重启后数据仍存在。
- 手机端可以通过同一 Wi-Fi 下的扫码或配对链接访问任务列表。
- 手机端可以查看、添加、完成和修改提醒时间，数据与电脑端一致。
- 局域网访问需要一次性配对 token，未配对设备不能直接读写任务。
- 项目包含可运行说明、核心测试和手动验证步骤。

## Constraints

- 项目当前是 AI Dev Team Starter Kit 文档仓库，没有现成应用代码。
- 默认使用 Simplified Chinese 与用户沟通，代码、命令、文件名和 API 名保留英文。
- 第一版应优先选择小而完整的本地优先架构，避免提前引入云端复杂度。
- 桌面端推荐使用 `Tauri`，前端推荐使用 `React` + `TypeScript`，本地存储使用 `SQLite`。
- 手机端采用响应式 Web 页面，复用核心任务 API 和 UI 逻辑。

## Open Questions

- 是否需要同时支持 Windows 和 macOS 打包，还是第一版先以当前 Windows 开发环境为准。
- 系统通知的提醒样式是否需要支持声音、置顶或重复弹出；第一版默认做普通系统通知和稍后提醒。
